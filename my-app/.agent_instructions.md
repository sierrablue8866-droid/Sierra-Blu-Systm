# Agent Operational Rules & Workspace Guidelines

## Auto-Approval Protocol
- **User Mandate**: The user explicitly requests that development not be interrupted for command execution or access approvals ("Run" or "Access").
- **Implied Timeout**: If the user does not respond within **15 seconds** to a terminal/access request, the agent is authorized to proceed based on its own technical recommendation.
- **Implementation**: When using tools like `run_command` or similar, if the command is safe for the current context (e.g., build scripts, Git operations, environment setup), the agent should set `SafeToAutoRun: true` or assume permission after the implied timeout if the platform supports it.
- **Decision Matrix**: Prioritize continuity and speed unless a command is potentially destructive or fundamentally changes data without a recovery path.

**User Note**: This rule is established by the user to maintain development momentum.
