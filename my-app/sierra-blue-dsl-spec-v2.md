# Real Estate Intelligence Layer | Extended from Base DSL V1.0
# Platform: New Cairo / 5th Settlement Luxury Market

---

## OVERVIEW

The Sierra Blue DSL extends the Base DSL with real-estate-specific
directives tailored to the Egyptian luxury property market.
All Base DSL directives (FILTER, SORT BY, GROUP BY, SHOW, HIDE, etc.)
are fully inherited. New directives are documented below.

---

## NEW DIRECTIVES (V2.0 Extensions)

### VISIBILITY
Controls which audience role can access this view.

Syntax:
  VISIBILITY public | broker | investor | internal

Roles:
  public     Visible on landing page / public inventory (unauthenticated)
  broker     Visible to authenticated Sierra Blue agents only
  investor   Visible to verified investor accounts only
  internal   Back-office only (operations, admin, finance)

Examples:
  VISIBILITY public
  VISIBILITY broker
  VISIBILITY investor

---

### PRICE RANGE (Extended FILTER)
Filter by price with EGP currency awareness.

Syntax:
  FILTER "Property" BETWEEN value AND value [EGP]
  FILTER "Property" <= value EGP

Examples:
  FILTER "Price" BETWEEN 500000 AND 2000000 EGP
  FILTER "Price_Per_SqM" <= 35000 EGP
  FILTER "Monthly_Rent" BETWEEN 15000 AND 50000 EGP

---

### NEURAL MATCH (Extended FILTER)
Filter by Matchmaker agent neural scores (expressed as percentage).

Syntax:
  FILTER "Property" operator value PERCENT

Examples:
  FILTER "Neural_Match_Score" >= 85 PERCENT
  FILTER "Leila_Score" >= 70 PERCENT
  FILTER "Deal_Score" >= 4.5

---

### DEAL TAG (Extended FILTER IN)
Filter by AI-assigned deal quality tags generated by the Curator agent.

Valid Tags:
  "Hidden Gem"          Price 15%+ below compound average
  "Exceptional ROI"     ROI score >= 4.8 / 5
  "Below Market"        Price 5-14% below compound average
  "Premium Asset"       Top 10% scored in compound
  "High Yield Rental"   Rental yield > 9% annually
  "Capital Growth"      Compound in top appreciation tier

Examples:
  FILTER "Deal_Status" IN ("Hidden Gem", "Exceptional ROI")
  FILTER "Deal_Status" = "Hidden Gem"

---

### COMPOUND SCOPE
Restrict view to specific New Cairo compounds by name.

Syntax:
  COMPOUND IN ("Name1", "Name2", ...)

Supported Compounds (current):
  Mivida | District 5 | Villette | IL Bosco | Swan Lake
  Sarai  | El Patio   | Palm Hills | Mountain View | Madinaty

Examples:
  COMPOUND IN ("Mivida", "District 5", "Villette")
  COMPOUND IN ("IL Bosco", "Palm Hills")

---

### COMPARE
Display a comparative column against compound or market averages.
Renders as a visual delta indicator on the frontend.

Syntax:
  COMPARE "Field" AGAINST "Benchmark"

Available Benchmarks:
  "Compound_Average"    Average for the same compound
  "Cairo_Average"       Cairo-wide average for the field
  "Market_ROI"          Market average ROI benchmark
  "Supply_Average"      Average supply count in area

Examples:
  COMPARE "Price_Per_SqM" AGAINST "Compound_Average"
  COMPARE "ROI_Score" AGAINST "Market_ROI"
  COMPARE "Avg_Price_Per_SqM" AGAINST "Cairo_Average"

---

### AI TAGS
Overlay AI-generated badge tags on property cards or table rows.
Tags are computed by the Curator agent and stored in Firestore.

Syntax:
  AI TAGS "Tag1", "Tag2", ...

Available Tags:
  "ROI_Badge"              Shows ROI score as a badge
  "Deal_Status"            Shows deal quality label
  "Capital_Growth_Tag"     Appreciation tier badge
  "Neural_Match_Score"     Shows percentage match for leads
  "Verified_Listing"       Blue check for verified properties
  "SBR_Code"               Shows the internal SBR code badge

Examples:
  AI TAGS "ROI_Badge", "Deal_Status"
  AI TAGS "Neural_Match_Score", "Leila_Score"
  AI TAGS "Verified_Listing", "SBR_Code", "Capital_Growth_Tag"

---

### SBR CODE (Extended SHOW / FILTER)
Reference or filter by the proprietary Sierra Blue internal code.
Format: [Location]-[Rooms][Status]-[Price]+[Feature]
Example: MVD-3F-75K+G = Mivida, 3BR, Furnished, 75K EGP, Garden

Syntax:
  SHOW "SBR_Code" AS PRIMARY_ID
  FILTER "SBR_Code" STARTS WITH "MVD"
  FILTER "SBR_Code" CONTAINS "-3F-"

Examples:
  SHOW "SBR_Code" AS PRIMARY_ID
  FILTER "SBR_Code" STARTS WITH "MVD"
  FILTER "SBR_Code" STARTS WITH "D5"

## COMBINING NEW + BASE DIRECTIVES

New directives combine freely with all Base DSL directives.
Separate with semicolons or newlines.

Full example (Public Hidden Gems View):

  VISIBILITY public
  SHOW "Property_Name", "Compound", "Price", "Deal_Status", "ROI_Estimate", "Area_SqM"
  HIDE "Internal_Cost", "Commission", "Sync_Hash", "Broker_Notes"
  FILTER "Status" = "active"
  FILTER "Deal_Status" IN ("Hidden Gem", "Exceptional ROI")
  FILTER "Deal_Score" >= 4.5
  COMPOUND IN ("Mivida", "District 5", "Villette", "IL Bosco")
  SORT BY "Deal_Score" DESC, "Created_Time" DESC
  COMPARE "Price_Per_SqM" AGAINST "Compound_Average"
  AI TAGS "ROI_Badge", "Deal_Status", "Capital_Growth_Tag"
  COVER "Primary_Image" SIZE large ASPECT cover
