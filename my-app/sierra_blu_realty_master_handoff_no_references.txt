SIERRA BLU REALTY - MASTER PROJECT HANDOFF

PROJECT IDENTITY
Brand Name: Sierra Blu Realty
Tagline: Beyond Brokerage
Official Description: AI-Powered Real Estate Intelligence

PROJECT DEFINITION
Sierra Blu Realty is not intended to be only a marketing landing page or a basic brokerage website. The intended direction is a luxury real estate operating system that combines premium brand presentation, structured brokerage operations, internal content control, curated inventory management, media control, and AI-assisted explanation layers.

The project should feel like a serious premium real estate platform built for trust, clarity, and long-term operational use.

CORE PRINCIPLES
1. Firestore is the source of truth.
2. AI explains, code decides.
3. Refactor before replace.
4. No heavy 3D in the MVP.
5. Public-facing website must feel premium and luxury.
6. Admin dashboard must be clean, practical, and operational.
7. Content editor is a core feature, not optional.
8. Real media and curated content matter more than placeholder visuals.
9. Public luxury screens must not become generic template UI.
10. Integration logic must preserve editorial overrides and avoid destructive overwrites.

HERO RULES
The hero must support three modes:
1. Lightweight Spline mode
2. Layered mouse-parallax mode
3. Static fallback mode

Hero runtime rule:
- use Spline only when lightweight and performance-safe
- if reduced motion is active or Spline is not appropriate, use parallax
- if neither should run, use static fallback

No heavy 3D should be used in the MVP.
No hero concept should make the site feel like a game or a flashy concept site.

4. Firestore remains canonical even when external data is pulled in.
5. Sync must be official integration logic, not scraping.

MAIN RISKS
The most important current risks are:
1. Selecting the wrong repo before canonical decision is finalized.
2. Over-reusing generic system UI on premium public pages.
3. Keeping Spline active even when performance suffers.

5. Delaying Firestore wiring too long and leaving the project stuck in shell mode.
6. Allowing provider sync to overwrite manual or editorial work.
7. Not having a safe manual review path for dedupe ambiguity.
